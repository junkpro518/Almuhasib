# المحاسب — Almuhasib Bot — Design Spec

**Date:** 2026-06-12  
**Status:** Approved

---

## Overview

A Telegram bot that receives bank purchase notifications from iPhone Shortcuts, lets the user mark transactions as "under the tab" (تحت الحساب) for a single person, stores them in Notion, and generates PDF reports on demand.

---

## Architecture

```
iPhone Shortcuts
      │
      │ HTTP POST (raw bank SMS text + secret key)
      ▼
  VPS Server
  ┌─────────────────────────────────┐
  │  Flask Webhook Endpoint (:8080) │  ← receives and parses SMS
  │  Telegram Bot (python-telegram-bot v20+) │  ← handles all user interaction
  │  Notion Client (notion-client)  │  ← stores/reads entries
  │  PDF Generator (fpdf2)          │  ← generates reports
  └─────────────────────────────────┘
      │
      ▼
  Telegram (user)
```

**Security:** The Flask endpoint requires a secret key in the request header (`X-Secret-Key`). The bot only responds to the owner's chat ID (hardcoded in config).

---

## SMS Parsing

Input format from bank:
```
شراء عبر نقاط بيع SAR 10.50
بطاقة 7796* مدى- ApplePay
من MATHAF ALGHIDHA EST
في 21:41 26-06-13
```

Extracted fields via regex:
- `amount`: `10.50` (float)
- `card`: `7796* مدى- ApplePay`
- `merchant`: `MATHAF ALGHIDHA EST`
- `datetime`: `2026-06-13 21:41`

## Bot Interaction Flow

### Incoming Bank Transaction

Bot sends to user:
```
💳 عملية شراء جديدة

المتجر:  MATHAF ALGHIDHA EST
المبلغ:  SAR 10.50
البطاقة: 7796* مدى - ApplePay
التاريخ: 2026-06-13  الساعة 21:41

هل هذه العملية تحت الحساب؟

[ ✅ نعم ]  [ 📝 نعم + ملاحظة ]  [ ❌ لا ]
```

**Button behavior:**
- `✅ نعم` → save to Notion immediately, edit message to show confirmation
- `📝 نعم + ملاحظة` → bot replies "أرسل ملاحظتك:", waits for user text, then saves to Notion
- `❌ لا` → edit message to show "تم التجاهل ✓", no save

### Manual Entry (`/add`)

Conversational flow — bot asks each field in sequence:

1. Bot: `كم المبلغ؟ (سالب للخصم، مثال: -30)`
2. User: `50`
3. Bot: `ما المتجر أو الوصف؟`
4. User: `مطعم الكوفي`
5. Bot: `ملاحظة إضافية؟ (أرسل /skip للتخطي)`
6. User: `/skip` or free text
7. Bot: `✅ تم الحفظ` + summary

## Commands

| Command | Description |
|---------|-------------|
| `/report` | Generate PDF of all entries + total, send as file |
| `/clear` | Delete all entries from Notion (with confirmation prompt) |
| `/add` | Start guided manual entry conversation |

---

## Notion Database Schema

Database name: **المحاسب**

| Column | Type | Notes |
|--------|------|-------|
| التاريخ | Date | Transaction date/time |
| المتجر | Title | Merchant name or description |
| المبلغ | Number | SAR amount (negative for deductions) |
| ملاحظة | Text | Optional note |
| النوع | Select | "بنكي" or "يدوي" |
| البطاقة | Text | Card info (bank entries only) |

---

## PDF Report Format

- **Title:** حساب المدفوعات (centered, Arabic)
- **Subtitle:** تاريخ التقرير: YYYY-MM-DD
- **Table columns:** # | التاريخ | المتجر | المبلغ (SAR) | ملاحظة
- **Styling:**
  - Arabic headers (RTL), data as-is
  - Negative amounts in red
  - Total row at bottom
- **Font:** Arabic-capable font (e.g., Amiri or Cairo) for Arabic headers
- **Generated by:** fpdf2 + arabic-reshaper + python-bidi

## iPhone Shortcuts Setup

The Shortcut triggers on bank notification arrival:
1. Receives notification text
2. Sends HTTP POST to `http://<VPS_IP>:8080/transaction`
3. Headers: `X-Secret-Key: <secret>`, `Content-Type: application/json`
4. Body: `{"text": "<raw SMS text>"}`

---

## Error Handling

- Unparseable SMS → bot notifies user with raw text, no buttons
- Notion API failure → bot notifies user, logs error
- Invalid `/add` amount (non-numeric) → bot re-asks
- `/clear` requires typing `تأكيد` to proceed (prevents accidents)

---

## Deployment

- VPS: Linux (Ubuntu 22.04+)
- Process manager: systemd or PM2
- Both Flask (port 8080) and Telegram bot run in the same Python process
- Bot uses polling (not webhook) to avoid SSL cert complexity
